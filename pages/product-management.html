🍞 商品データベース管理

事務作業者向け - 商品マスターの管理業務完全ガイド

📋 商品データベースの重要性

商品データベースはシステム全体の核となるデータです。
ここに登録された情報が、製造表・発注表・ラベル・明細書など、すべての業務に影響します。
正確な登録と管理が、円滑な業務運営の基盤となります。

📊 商品データの項目詳細

🔍 各項目の役割と入力のポイント

システムが正しく動作するために、各項目の意味と正しい入力方法を理解しましょう。

🏷️

基本情報項目

項目名 説明 入力例 注意点
商品ID カテゴリ頭文字＋4桁数字 S0001, B0025 一度設定したら変更禁止
商品名 商品の正式名称 メロンパン, 発酵バタークロワッサン "レシピ"シートと完全一致必須
売価 ローゼンでの販売価格 180, 250 レシピ、ラベルと完全一致必須
クリエ売価 クリエイト店舗での販売価格 150, 200 ラベルと完全一致必須
🏷️

ラベル・識別項目

項目名 説明 入力例 注意点
ラベル ローゼンラベル印刷用番号 5015, 6125 4000以降の番号、ラベルと完全一致必須
クリエラベル クリエイト用ラベル番号 8002, 8015 8000以降の番号、ラベルと完全一致必須
カテゴリ 商品の種類分類 食パン, 菓子パン, ドーナツ, デニッシュ 統一された表記で入力
金探 金属探知機登録番号 031, 012 金属探知機の設定番号と一致
⚙️

製造関連項目

項目名 説明 入力例 注意点
期限 消費期限（納品日からの日数） 2, 3, 1 日数で入力（当日なら0）
生地名 使用する生地の種類 アズキレンニュウ, フランスキジ 生地出し表に表示される名前
並べる表 並べる表への表示 ○,空白 ○なら並べる表に表示、空白なら表示しない
生地出し 生地出し表への表示 ○, ×, 空白 冷凍生地は○、スクラッチ生地は×、空白なら表示しない
📦

数量・包装項目

項目名 説明 入力例 注意点
生地数 1個を作るのに必要な生地個数 1, 2, 0.5 食パン1/2でも1/4でも"1"
何個入 1パッケージの入数 1, 2, 0.5 食パン1/2本なら0.5
最大数 1天板に乗せられる最大個数 12, 20, 8 パンのサイズに応じて設定
倍数 並べる際の倍数 5, 10, 1 ミニクロなど、5個ずつ並べるなら"5"
袋 使用する包装材の種類 防曇袋, 1斤袋, ニラ袋 発注表で材料として計算
📋

管理項目

項目名 説明 入力例 注意点
終売 販売終了商品のマーク ○, 空白 ○を入れると表に表示されない

✨ 新商品登録手順

1

事前準備

新商品登録に必要な情報を整理します

📋 必要な情報チェックリスト

  • ☐ 商品名（正式名称）
  • ☐ 販売価格（自社・他社）
  • ☐ カテゴリ分類
  • ☐ 消費期限（日数）
  • ☐ 使用する生地の種類
  • ☐ 包装仕様（袋の種類）
  • ☐ 製造仕様（生地数、天板最大数など）
  • ☐ ラベル印刷番号
  • ☐ 金属探知機設定番号(あれば)
⚠️ 不明な項目がある場合は、確認してから登録
2

商品IDの決定

カテゴリに応じた一意の商品IDを決定します

🏷️ 商品IDの命名規則

カテゴリ 頭文字 例
食パン S S0001, S0002...
バケット B B0001, B0002...
ロール R R0001, R0002...
菓子パン K K0001, K0002...
調理パン C C0001, C0002...
ドーナツ D D0001, D0002...
デニッシュ E E0001, E0002...
その他 O O0001, O0002...

📋 ID決定手順

  1. 該当カテゴリの最新番号を確認
  2. 次の連番を決定
  3. 商品データベースで重複していないか確認
  4. 決定したIDをメモ
🚨 商品ID重複は絶対禁止！必ず確認
3

商品データの入力

商品データシートに新商品の情報を入力します

📝 入力手順

  1. バックアップ作成
    登録前に必ずファイルのバックアップを作成
  2. 商品データシートを開く
    「商品データ」シートを選択
  3. 最下行に移動
    既存データの最後の行の次に移動
  4. 項目ごとに入力
    左から順番に、各項目を正確に入力
  5. 入力内容の確認
    全項目を再度チェック
  6. まとめて出力
    "操作"シートの全部まとめて出力ボタンを押して、出力された表を確認

⚠️ 入力時の注意点

  • 商品名に使う記号やスペースは半角で入力
  • 数値項目に文字が混入していないか
  • ○×項目は正確に入力されているか
  • 既存のカテゴリ・生地名と表記が一致しているか確認
💡 入力後は保存を忘れずに
4

関連データの登録

新商品に関連する他のデータも合わせて登録します

🔗 関連登録項目

  • レシピシート
    新商品の材料と使用量を登録
  • 各店舗シート
    取り扱い店舗の納品個数を設定(取り扱いが無くても0を入力)
  • 材料マスター
    新規材料がある場合は追加
✅ 関連データも同時に整備することで、システム全体の整合性を保つ
5

動作確認

新商品が正しく各表に反映されるか確認します

🔍 確認項目

  • 製造数表に新商品が表示されるか
  • 生地出し表で計算されるか
  • 並べる表に表示されるか（○設定の場合）
  • 発注表で材料が計算されるか
  • ラベル出し表に表示されるか

📋 確認手順

  1. 「操作」シートから更新ボタンを実行
  2. 各表で新商品の表示を確認
  3. エラーが発生していないか確認
  4. 計算結果が正常か確認
⚠️ エラーがある場合は商品データを見直し

📴 終売商品の処理

📋

終売設定の実施

販売終了となった商品を表示しないように設定します

🔧 終売設定手順

  1. 商品データシートを開く
  2. 終売商品を検索
    Ctrl+Fで商品名を検索
  3. 「終売」列に○を入力
    該当商品の終売列に「○」を入力
  4. 保存
    変更内容を保存
  5. 動作確認
    各表から商品が消えることを確認

⚠️ 終売設定の注意点

  • 完全削除ではなく、非表示処理
  • 過去のデータは保持される
  • 間違えて設定した場合は○を削除すれば復活
  • 関連するレシピデータはそのまま残る
🔄 終売設定後は必ず更新ボタンを実行
🗂️

関連データの整理

終売商品に関連するデータの整理を行います

📁 整理対象

  • 店舗別納品個数
    各店舗シートで個数を0に設定
  • レシピデータ
    当面は保持（将来的に類似商品で再利用可能）
  • 材料マスター
    他商品で使用していない材料は要検討
📊 データの完全性を保ちながら、業務効率を向上

✏️ 商品情報の変更

💰

価格変更

売価・クリエ売価の変更

  1. 該当商品を検索
  2. 売価またはクリエ売価を修正
  3. 伝票・ラベル出しへの影響を確認
💡 価格変更は即座に反映
📅

期限変更

消費期限日数の変更

  1. 商品データの期限列を修正
  2. ラベル出し表の期限計算を確認
  3. 店舗への周知を忘れずに
⚠️ 食品安全に関わる重要変更
📦

包装・仕様変更

袋・生地・製造仕様の変更

  1. 該当項目を修正
  2. 発注表への影響を確認
  3. 製造現場への情報共有
🔄 関連表の自動更新を確認

📞 店舗からの変更依頼対応

📋

個数変更依頼の処理

店舗から納品個数変更の依頼があった場合の対応手順

📞 変更依頼の受付

  1. 依頼内容の確認
    店舗名、商品名、変更前後の個数、適用期間
  2. 変更理由の聞き取り
    一時的な変更か恒久的な変更かを確認
  3. 変更可能日の調整
    製造スケジュールとの調整
  4. 依頼内容の記録
    日時、担当者、変更内容を記録

✏️ システムでの変更作業

  1. 該当店舗シートを開く
    店舗マスターから該当店舗のシートに移動
  2. 商品と日付を特定
    変更対象の商品と日付を確認
  3. 個数を変更
    新しい個数を入力
  4. 製造数への影響確認
    製造数シートで合計が変わることを確認
  5. 更新ボタン実行
    関連表を最新状態に更新
⚠️ 大幅な変更は製造・発注に影響するため、関係部署に連絡

🏆 データ管理のベストプラクティス

💾

バックアップ管理

  • 商品データ変更前は必ずバックアップ
  • 日付入りファイル名で管理
  • 最低1週間分は保持
  • 重要変更時は複数箇所に保存
🚨 バックアップは業務継続の生命線
📝

変更記録の管理

  • 変更日時と担当者を記録
  • 変更理由を明記
  • 影響範囲を文書化
  • 承認者の記録
📋 変更履歴は品質管理の基本
🔍

定期的な データ監査

  • 週1回のデータ整合性チェック
  • 重複データの確認
  • 未使用商品の洗い出し
  • エラーデータの修正
✅ 定期監査でシステムの健全性を維持
👥

情報共有の徹底

  • 新商品情報の共有
  • 価格変更の事前通知
  • 終売予定の早期アナウンス
  • システム変更の周知
📢 情報共有不足はトラブルの元

🛠️ よくあるエラーと対処法

❌

商品IDの重複エラー

症状：同じ商品IDが複数存在してシステムが正常動作しない

🔧 対処法

  1. 商品データシートで商品名でソート
  2. 重複している商品IDを特定
  3. どちらが正しいデータか確認
  4. 不要な方のIDを変更または削除
  5. 関連データの整合性を確認
🚨 重複IDは即座に修正が必要
🔤

商品名の不一致エラー

症状：レシピや発注表で商品が正しく表示されない

🔧 対処法

  1. 商品データベースの正確な商品名を確認
  2. レシピシートの商品名と比較
  3. スペース、文字種（ひらがな/カタカナ）の違いをチェック
  4. 不一致箇所を統一
  5. エラーが解消されることを確認
🔍 文字の微細な違いに注意
📊

計算結果の異常

症状：製造数や発注量が明らかにおかしい

🔧 対処法

  1. 該当商品の基本データをチェック
  2. 生地数、何個入、倍数の設定を確認
  3. 単位の間違い（kg/g）がないか確認
  4. コードの参照先が正しいか確認
  5. 手計算で検算
🧮 疑問があれば必ず検算

🆘 商品データベース管理の重要事項

商品データベースはシステム全体の基盤です。
変更作業は慎重に行い、不明な点は必ず確認してください。

データの整合性が崩れると、製造・発注・配送すべてに影響します。
「分からない時は聞く」「確信がない時は確認する」を徹底しましょう。
